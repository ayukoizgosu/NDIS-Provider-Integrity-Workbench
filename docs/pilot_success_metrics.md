# Pilot Success Metrics

## Purpose

These metrics define whether the environment is useful for analyst-assisted public-record review.

They are intentionally operational and measurable.

## Core Outcomes

The environment is successful if analysts can:

- find a record quickly
- understand the evidence presented
- create and reopen a case
- add notes and a decision trail
- export a readable brief
- keep review items visible rather than hidden

## Suggested Metrics

### Search Efficiency

- time from opening the app to first useful lookup
- time from lookup to opening a record
- number of searches completed per session

### Case Workflow

- number of cases created
- number of cases updated
- number of notes added per case
- number of cases exported as briefs
- time from lookup to case creation

### Review Quality

- percentage of records that remain reviewable
- percentage of cases with at least one analyst note
- percentage of cases with source links attached
- percentage of case briefs exported without manual cleanup

### Manager Usefulness

- number of open cases by status
- number of high-priority cases
- number of escalated cases
- average age of open cases

### Evidence Quality

- count of records with a confirmed public business match
- count of records that require state check or human review
- count of related-business leads generated from public data

## Demo Acceptance Thresholds

For a small deployment, the following are reasonable acceptance thresholds:

- lookup and record opening work reliably in the browser
- a case can be created in one step from a lookup result
- a case can be updated and reopened after refresh
- notes and sources persist in the case log
- exported briefs are readable by a non-developer

## What Not To Measure

Do not claim success based on:

- fraud detection accuracy
- automated case outcomes
- director-overlap confirmation
- internal integration coverage

Those are outside the current scope.

## Example Buyer Value Statements

- analysts spend less time assembling evidence
- supervisors get a cleaner queue and clearer case summaries
- public records are easier to review consistently
- lookup-to-brief turnaround is reduced
